# Week 4: Three Roads (Choose Your Stack with AI)

## 🤖 The AI Prompt Setup
I fed an AI my exact constraints:
1. **Budget:** Free only.
2. **Skill Level:** Backend Intern (strong in Node.js/Python, weak in complex frontend/React).
3. **What it needs to do:** Host a Single Page Application (SPA) with 4 sections: Hero, Background Jobs Case Study, Web Scraper Case Study, and Bio/Contact.
4. **How it must be displayed:** Must easily display raw terminal screenshots, embedded Vercel dashboard links, and GitHub repo links. No dynamic form handling or database needed yet.

---

## 🛣️ The Three Options Generated by AI

1. **The Simplest: Vanilla HTML/CSS on Vercel**
   - *How to build:* Write pure HTML and CSS using the predefined Identity Kit. No build steps.
   - *Hosting:* Vercel (Free tier).
   - *Backend needed?* No.
   - *Trade-off:* Harder to scale if I want to add 20 case studies later, as I'd have to copy/paste HTML blocks manually instead of using components.

2. **The Middle Ground: Astro (Static Site Generator) on Vercel**
   - *How to build:* Write content in Markdown (`.md` files) and let Astro generate the HTML. 
   - *Hosting:* Vercel (Free tier).
   - *Backend needed?* No.
   - *Trade-off:* Requires learning Astro's specific syntax and setting up a build step, which might distract from getting the content out.

3. **The Most Powerful: Next.js + React on Vercel**
   - *How to build:* Build a full React application with component-based architecture.
   - *Hosting:* Vercel (Free tier).
   - *Backend needed?* Optional, but supports serverless functions.
   - *Trade-off:* Extreme overkill for a single-page portfolio. Massive learning curve for a backend dev just trying to show off Python/Node scripts.

---

## 🧠 The Pressure Test

I pressure-tested the "Simplest" option (Vanilla HTML/CSS) against my goals:
- *What breaks if I pick the simplest?* Nothing. It's a single page. If I had 15 pages, maintaining a universal nav bar would break without a framework, but for an SPA, it's perfect.
- *Does it show my work well?* Yes. `<img src="...">` and `<a>` tags display terminal screenshots and repo links flawlessly without needing an image optimization library.
- *Can I finish in two weeks?* Absolutely. There are zero compilation errors to fight.

---

## 🔨 My Final Rationale

**I am choosing Option 1: Vanilla HTML/CSS hosted on Vercel.**

**Why I chose it:** I am a backend intern focused on data pipelines and job queues. My portfolio's job is to prove I can write robust backend logic, not to prove I can build complex React frontend architectures. Vanilla HTML/CSS is completely free, guarantees I can finish the build in the next two weeks without fighting frontend frameworks, and perfectly displays my terminal screenshots and case studies. 

**Why I rejected the others:** 
Next.js (Option 3) is massive overkill. If I pick Next.js, I will spend the next week fighting React hooks and deployment caching instead of writing my case studies. I also rejected Astro (Option 2) because while Markdown is nice, setting up a static site generator for a single-page website introduces a build step I simply don't need yet. 

**Can I maintain this?** Yes. Maintaining a single `index.html` file requires zero dependencies. If I look at this code 3 years from now, it will still work perfectly without needing `npm install` or dealing with deprecated libraries. And no, I do not need a backend for this portfolio—my work *is* the backend, and the portfolio is just the static frame around it.
